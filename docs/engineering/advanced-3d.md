# 3D 进阶能力应该从哪里开始

主线 3D 文档已经覆盖了场景搭建、相机与光照、模型渲染、动画接入和交互拾取。再往下走，通常会碰到一组被笼统称为"进阶"的能力：

1. 粒子系统
2. 拖尾
3. 后处理
4. 自定义 Shader
5. NavMesh 寻路

这些能力之间没有强依赖，但学习顺序混乱时，最常见的结果是：一个还没做完，就跳去碰另一个，最后每个都停在"能跑起来但说不清"的状态。

这篇不展开每个能力的细节，只回答一个问题：**作为一个还没碰过这些进阶能力的人，应该按什么顺序、带着什么前提去开始。**

## 先确认前置基础是否已经稳了

进阶能力不是独立模块，它们都建立在主线 3D 知识之上。在开始之前，至少先确认下面几件事你已经能稳定做到：

1. 能用 `Scene3D`、`Camera`、`Sprite3D` 搭出一个能正确显示的 3D 场景。
2. 理解 `Transform3D` 的位置、旋转、缩放，以及父子层级的变换继承。
3. 能加载模型预制体并实例化到场景里。
4. 理解材质、纹理和渲染器的基本关系。
5. 能用脚本驱动 3D 节点的每帧行为。

如果这几条还没站稳，先回到主线 3D 文档补齐。进阶能力会放大主线里的任何模糊点——例如一个粒子挂错了层级，或者一个 Shader 用错了变换矩阵，问题通常都不在进阶 API 本身，而在前置的节点和变换理解上。

## 五项能力各自解决什么问题

在决定先学哪个之前，先分清它们各自在解决哪一类需求：

| 能力 | 解决的核心问题 | 典型场景 |
|------|----------------|----------|
| 粒子系统 | 大量短生命周期视觉元素的整体表现 | 火焰、烟雾、爆炸、魔法特效 |
| 拖尾 | 一个运动物体沿轨迹留下的连续带状视觉 | 武器挥动残影、子弹尾迹、赛车光带 |
| 后处理 | 对整张渲染结果做画面级加工 | 泛光、色调映射、景深、抗锯齿 |
| 自定义 Shader | 控制单个材质的逐像素渲染逻辑 | 水面、溶解、卡通描边、特殊材质 |
| NavMesh 寻路 | 让角色在可行走区域内自动绕障移动 | NPC 巡逻、点击移动、AI 路径规划 |

这张表的价值不是分类本身，而是帮你在面对一个具体需求时，先判断它属于哪一类，再决定往哪个方向深入。

## 推荐的学习顺序

这五项能力没有唯一正确的顺序，但下面这条路径对大多数人来说踩坑最少：

### 第一步：粒子系统

理由：

1. 粒子是这五项里最"所见即所得"的，反馈周期短。
2. 粒子资源通常以预制体或特效资源形式存在，加载和使用方式和普通模型接近。
3. 粒子能让你快速熟悉"特效对象如何进入 3D 场景树"这件事，这是后续拖尾和其他特效类能力的共同基础。

最小起点是理解一个粒子特效如何被加载并挂到场景或某个节点上：

```ts
const fxPrefab = await Laya.loader.load("resources/3d/fx/Hit.lh") as Laya.Prefab;
const fx = fxPrefab.create() as Laya.Sprite3D;
scene.addChild(fx);
```

这一步的核心不是粒子参数怎么调，而是确认你已经能稳定地把一个特效对象放进场景、定位到正确位置、在合适时机销毁。

### 第二步：拖尾

拖尾和粒子在视觉表现上接近，但实现思路不同：

1. 粒子是大量独立元素的集合。
2. 拖尾是跟随单个运动物体的一条连续带。

拖尾适合放在粒子之后学，因为它需要你先理解"一个特效对象如何挂到某个运动节点上并跟随它"。如果这一步在粒子里已经练过，拖尾的接入成本会低很多。

最小起点是理解拖尾组件如何附加到一个 `Sprite3D` 上，并随其运动产生带状视觉。

### 第三步：后处理

后处理和前两项有一个本质差别：**它作用的对象不是某个节点，而是整张画面。**

这意味着在学后处理之前，你需要先对"相机输出的是什么"有一个基本认识。主线文档里关于相机和渲染的部分如果已经站稳，后处理的入口就不会太突兀。

后处理的学习重点通常不是"怎么开"，而是：

1. 后处理效果之间会互相影响，叠加越多越难控制性能和画面一致性。
2. 移动平台和小游戏平台上，后处理往往是性能瓶颈的第一来源。
3. 先学会有节制地用一两个效果，再考虑组合。

### 第四步：NavMesh 寻路

NavMesh 放在后面，是因为它和前三项的差异最大：

1. 前三项都是视觉表现层的能力。
2. NavMesh 是逻辑和 AI 层的能力，它解决的是"角色该往哪走"。

NavMesh 的前置不只是 3D 基础，还需要你对"角色如何移动"有一套自己的方案——例如是用 `Transform3D` 直接改位置，还是用物理刚体驱动。如果角色移动本身还没理清，寻路效果会很难判断对不对。

NavMesh 通常涉及两件事：

1. 场景的可行走区域如何生成和表示。
2. 运行时如何根据目标点查询路径并驱动角色沿路径移动。

建议在已经有了一个能手动移动的角色之后，再接入 NavMesh 把"手动移动"替换成"自动寻路"。

### 第五步：自定义 Shader

自定义 Shader 放最后，不是因为它最不重要，而是因为它对前置知识的要求最综合：

1. 需要理解材质和渲染管线。
2. 需要理解顶点变换、UV、法线这些图形学基础。
3. 需要理解引擎的渲染流程和 Shader 接入方式。

如果你在没站稳前四项的情况下直接跳到 Shader，最常见的结果是：效果能抄出来，但出了问题完全不知道该查哪一层——是矩阵传错了、是材质参数没生效、还是渲染阶段本身没走到。

自定义 Shader 是这五项里天花板最高、也最容易长期投入的一项。把它放在最后，意味着前面四项的经验都会成为它的支撑。

## 一个常见的节奏误判

进阶能力最容易出问题的不是某一项本身太难，而是节奏失控：

1. 同时开两三项，每项都只做到"能跑"。
2. 把进阶能力的接入和理解混在一起，导致每一项都停在表面。
3. 因为某一项遇到性能问题，就跳到另一项回避，结果几项都没真正落地。

更稳的做法是按上面的顺序，一次只深入一项，把它做到能在真实场景里稳定使用、能说清楚它的接入方式和常见问题，再开始下一项。

## 先从哪里写第一行代码

如果你已经确认前置基础稳了，并且决定从粒子系统开始，那么第一步可以非常小：

1. 准备一个已经能正确显示的 3D 场景。
2. 准备一个粒子特效资源（通常是 `.lh` 预制体）。
3. 用 `Laya.loader.load()` 加载它。
4. 用 `create()` 实例化后挂到场景或某个节点上。
5. 确认它能正确显示、能被移动和销毁。

就这一步走通，你已经跨过了进阶能力的入口。后面的拖尾、后处理、NavMesh、Shader，都是在"把一个进阶对象正确接入场景"这个相同模式上的扩展，只是每一项接入的对象类型和控制粒度不同。

## 小结

3D 进阶能力不是一组需要同时攻克的清单，而是一组可以按需分阶段进入的扩展能力。开始之前先确认主线 3D 基础已经稳了；开始之后按"粒子 → 拖尾 → 后处理 → NavMesh → Shader"的顺序逐项深入，一次只推进一项。这个顺序的核心理由不是难度排序，而是前置依赖和知识复用——每一项的能力都会成为下一项的支撑。
