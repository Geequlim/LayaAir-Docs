# 常见问题与排错索引

这篇文档是一个排错索引，不是完整教程。它的作用是：当你遇到某个现象时，能快速定位最可能的原因和该往哪个方向查。

每条只给三件事：**现象**、**可能原因**、**排查方向**。详细原理在对应主题文档里，这里不重复。

排查时先记住一条主线：大多数“代码写对了却不对”的问题，根因不在 API 本身，而在下面四条边界的某一条上：

1. 时机边界：初始化前还是初始化后
2. 链路边界：资源是加载完了，还是运行时对象已可用
3. 激活边界：节点在不在有效层级，组件是不是启用
4. 注册边界：类、loader、模块有没有真的进入运行时

## 运行时

### 引擎没反应，舞台上一片空白

- 现象：调用代码执行了，但屏幕什么都不显示
- 可能原因：`Laya.init()` 还没完成，就访问了 `Laya.stage` 或开始加载资源
- 排查方向：确认所有访问 `Laya.stage`、`Laya.loader` 的代码都在 `await Laya.init(...)` 之后

### 对象创建了但画面上看不到

- 现象：`new` 了一个显示对象，设置完属性却没出现
- 可能原因：没有 `addChild` 进显示树，或没挂到 `Laya.stage` 这棵树的某个节点下
- 排查方向：沿着父链往上追，确认最终能到达 `Laya.stage`

### 改了 `Config` 的渲染参数没生效

- 现象：设置了 `useWebGL2`、`isAntialias`、`fixedFrames` 等，结果没变化
- 可能原因：这些是初始化期参数，放在 `await Laya.init()` 之后修改不会按预期生效
- 排查方向：把 `Laya.Config.xxx = ...` 全部移到 `Laya.init()` 调用之前

### `Laya.stage.width` 在不同设备上行为不一致

- 现象：同样的布局代码，有的设备对、有的设备错
- 可能原因：`scaleMode` 不同，舞台宽高的语义就不同。`full`、`fixedwidth`、`fixedheight`、`fixedauto` 会让 `stage.width`/`height` 跟着设备变；`noscale`、`showall`、`noborder` 通常保持设计宽高语义
- 排查方向：先确认当前 `scaleMode`，再判断布局是否依赖会变化的 `stage.width`/`height`；依赖就监听 `Laya.Event.RESIZE` 重排

### 点击位置和视觉位置对不上，或边缘按钮点不到

- 现象：画面缩放、留边后，点击命中错位；`noborder` 下边缘内容被裁
- 可能原因：把 `alignH`/`alignV` 当成了坐标原点改变；或用 `noborder` 把交互元素放到了设计稿边缘
- 排查方向：输入坐标会被换算回舞台坐标，正常不需手动处理；边缘元素在 `noborder` 下改用相对 `stage.width` 定位

### 横竖屏表现混乱

- 现象：方向没按预期旋转或适配
- 可能原因：把 `screenMode`（横竖方向）和 `scaleMode`（尺寸映射）混成一件事
- 排查方向：`screenMode` 先决定方向语义，`scaleMode` 再决定尺寸映射，分开确认

## 资源

### `loader.load()` 成功了，但结果用不了

- 现象：`await` 返回了，对象却不是预期的运行时对象
- 可能原因：把“资源加载完成”和“运行时对象已可用”混成一件事。层级资源（场景、预制体）`load()` 拿到的是可实例化资源，还要 `create()` 或 `Scene.open()` 才进入运行时对象阶段
- 排查方向：区分资源类型——普通贴图接近直接可用，图集要按小图路径再取结果，场景/prefab 要继续实例化

### 图集加载完，取小图取不到

- 现象：`load("xxx.atlas")` 成功，但 `getRes` 拿小图为空
- 可能原因：图集建立的是“小图路径到纹理结果的映射”，不是返回一张大图
- 排查方向：用小图逻辑路径 `Laya.loader.getRes("xxx/icon.png")` 取，而不是用 `.atlas` 返回值

### 某类资源加载结果不对或报类型错

- 现象：路径写对了，加载结果却不是预期类型
- 可能原因：对应资源类型的 loader 没注册成功，或传了不合适的类型参数
- 排查方向：确认对应模块已进入运行时并完成 loader 注册；注意 `IMAGE`（偏 2D 加载入口）和 `TEXTURE2D`（偏底层纹理资源）不是同一层概念

### 同一个资源第二次加载行为和第一次不一样

- 现象：再次加载更快、或结果状态变了
- 可能原因：缓存命中直接返回旧结果，或之前调用过 `clearRes` 把它清掉了
- 排查方向：检查资源生命周期——是否被 `clearRes` 影响过；并发请求会被去重到同一条加载任务

### 场景关了，资源好像还在 / 第二次打开场景更快

- 现象：`scene.close()` 后内存没立刻降，或重开场景变快
- 可能原因：场景生命周期（实例状态）和资源生命周期（缓存状态）是两条线，关闭场景不等于立即释放资源
- 排查方向：资源是否还在缓存、是否仍被其他对象引用，要单独看资源系统状态

### 发布后资源缺失，本地能跑线上报错

- 现象：动态加载的资源在发布后读不到
- 可能原因：文件没有被收进构建产物。`assets/` 下没被引用也没进入有效动态加载规则的文件，不会默认进包
- 排查方向：确认动态资源路径稳定并真的进入产物；不要把 `resources/` 当万能兜底目录

## 构建与发布

### 本地预览正常，构建后白屏或启动失败

- 现象：`layaair run` 能开，构建后打不开
- 可能原因：把预览成功当成发布成功。入口资源、动态资源或脚本没有进入产物
- 排查方向：按入口链 → 代码链 → 依赖链 → 资源链 → 路径链顺序排查；`run` 和 `build` 职责要分开

### 平台名或构建参数报错

- 现象：构建命令失败或产物不对
- 可能原因：沿用了旧项目的平台参数习惯
- 排查方向：先 `layaair build -l` 确认当前 CLI 支持的平台名，并在脚本/CI 里写明版本

### 构建目录里的修改下次构建被覆盖

- 现象：手改产物后临时能跑，重新构建失效
- 可能原因：把构建目录当长期手工维护目录
- 排查方向：所有修改回到源码和源工程，构建产物视为可重复生成的交付结果

### 换目标平台后缺图、缺脚本、动态资源打不开

- 现象：Web 正常，小游戏平台异常
- 可能原因：不同目标的入口形式、资源收集规则和路径语义不同
- 排查方向：确认首场景路径稳定、动态资源进产物、没混用旧路径习惯（如旧 `res/...`）

### 远程资源切 CDN 后路径失控

- 现象：一换域名或目录，业务代码到处要改
- 可能原因：业务代码里直接拼了完整 CDN URL，没有稳定逻辑路径前缀
- 排查方向：业务代码保留稳定逻辑路径，源站切换收敛到 `Laya.URL.basePath` 或集中配置层

### 新版上线后用户仍拿到旧缓存

- 现象：代码发新版，旧图集/场景还在
- 可能原因：版本策略是整包思维，变更资源没有产生可区分的访问地址
- 排查方向：按文件或资源组做版本失效（文件名改写、查询串或 `fileconfig` 版本映射），不要整批统一改大版本号

## 场景与脚本

### 场景能打开，但脚本方法不执行

- 现象：`Scene.open()` 成功，挂的脚本没反应
- 可能原因：脚本类没注册，或类注册了但脚本代码没进入最终运行时；也可能是绑定成功了但宿主节点没进入有效激活链
- 排查方向：三问分清——是绑定失败还是没进激活链？类有没有 `@regClass()` 注册？当前运行时代码里这个类真的存在吗？

### `runtime` 声明的类没生效

- 现象：节点树出来了，自定义运行时类没替换上
- 可能原因：`runtime` 恢复和脚本组件恢复是同一条“按数据恢复类”链路，类没注册或没进运行时都会导致失败
- 排查方向：和脚本丢失一起查，根因通常是运行时没能把序列化类型还原成真实类

### `onAwake()` 写的逻辑，重新启用时不执行

- 现象：对象停用再启用，初始化没重跑
- 可能原因：`onAwake()` 只在首次激活执行一次；重新启用只会重跑 `onEnable()`
- 排查方向：只需一次的初始化放 `onAwake()`，每次重新启用都要重置的状态放 `onEnable()`

### 脚本有时执行有时不执行

- 现象：同样的脚本行为不稳定
- 可能原因：混淆了 `active`、`activeInHierarchy`、`enabled` 三个开关；或父节点不激活导致子树退出激活链
- 排查方向：确认 `activeInHierarchy` 是否为 `true`、组件 `enabled` 是否启用；`removeChild` 只是移出树，对象还活着，`destroy` 才是终点

### 输入事件收不到

- 现象：点击/按键没响应
- 可能原因：输入先进入 `Laya.stage` 再进显示树；脚本里鼠标事件接 `owner`、键盘事件接 `Laya.stage`，监听对象搞错了
- 排查方向：对象级输入监听 `owner`，全局输入（键盘、`resize`）监听 `Laya.stage`；脚本约定方法会自动接好对应入口，确认宿主在激活链里

### 对象池复用后状态怪异

- 现象：复用的对象带着上次的残留状态
- 可能原因：把 `removeChild` 当成了 `destroy`，或重置逻辑放错了生命周期位置
- 排查方向：复用对象的重置放 `onEnable()`；确认没有在已销毁对象上继续操作

### 每帧逻辑或缓动停不下来

- 现象：`frameLoop`、`Tween` 在对象停用后还在继续改状态
- 可能原因：全局 `timer`/`Tween` 不天然依附节点生命周期，对象停用不会自动停掉外部调度
- 排查方向：让调度边界和对象边界一致——用 `this` 作为 `caller` 归属，或在 `onDisable()` 显式清理

## 怎么用这份索引

1. 先按现象归类到运行时、资源、构建、场景四块
2. 在对应块里找到最接近的条目
3. 顺着排查方向回到对应主题文档看原理，而不是在这里死磕

如果某条现象这里没有，先按开头那四条边界（时机、链路、激活、注册）自检一遍，通常能缩小到具体主题文档。
