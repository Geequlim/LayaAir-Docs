# 参考资料分级与取材策略

## 目的

这份表用于帮助后续写作者快速判断：

1. 哪些官方文档适合直接作为主线来源
2. 哪些文档只能抽取概念后重写
3. 哪些文档应延后处理

这是一份内部写作分级表，不是面向最终读者发布的引用索引。

后续如果把其中结论写进正式文档：

1. 不保留本地参考路径
2. 不在正文暴露内部资料组织方式
3. 只沉淀整理后的知识结论

## A 类：主线优先

这些内容与运行时能力或标准项目开发强相关，应优先进入主线文档。

1. 官方中文文档中的基础通用章节
2. 官方中文文档中的 2D 章节
3. 官方中文文档中的 3D 章节
4. 官方中文文档中的命令行发布章节
5. 官方中文文档中的通用发布设置章节

关注点：

1. 对象模型
2. 运行时能力
3. 资源加载方式
4. 构建与部署共性

## B 类：标题偏 IDE，但内容高价值

这些内容不能整篇搬运，但必须仔细读。

1. 项目目录说明类文档
2. 启动入口说明类文档
3. 预制体说明类文档
4. 材质与渲染资源说明类文档
5. 场景环境设置说明类文档
6. 自定义组件系统说明类文档
7. 装饰器与类注册说明类文档
8. 物理组件说明类文档
9. UI 运行时说明类文档

处理方式：

1. 提炼运行时概念
2. 拆回对应主线主题
3. 删除纯面板说明
4. 用代码和数据结构重新组织

## C 类：可作为工作流补充

这些内容不适合作为主线知识结构，但适合放进工程化或补充章节。

1. 开发环境相关文档
2. 开发工具相关文档
3. 发布相关文档

处理方式：

1. 与 CLI 真实能力合并
2. 按平台共性重写
3. 减少截图依赖

## D 类：延后或降级

这些内容偏编辑器使用手册，不应挤占主线优先级。

1. 创建项目向导类文档
2. 快捷键类文档
3. UI 编辑器基础操作类文档
4. 商店发布或包导出类文档

处理方式：

1. 暂不进入主线
2. 后续视需要整理为附录

## 源码使用策略

以下主题应优先以源码为准，再回看官方文档：

1. 引擎初始化
2. 生命周期
3. 资源加载
4. 层级资源解析
5. 预制体实例化
6. 资源释放
7. 模块注册和副作用导入

建议优先阅读：

1. 引擎初始化入口
2. 资源加载入口
3. 组件核心实现
4. 脚本核心实现
5. 层级资源加载实现
6. 层级资源解析实现
7. 资源生命周期核心实现

## 专项技能资料的定位

CLI / 插件类专项资料更像技能型资料库：

1. CLI 资料适合命令行与自动化章节
2. 插件资料适合插件和扩展章节

它不适合作为完整目录骨架，但很适合作为：

1. 命令示例来源
2. 限制条件来源
3. agent 友好的写法参考
