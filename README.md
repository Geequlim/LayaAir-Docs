# LayaAir 文档重写计划

这个项目的目标不是简单搬运 LayaAir 官方文档，而是在官方资料基础上补充更结构化、可检索、适合 Agent 使用的说明。

官方资料已经覆盖了大量 LayaAir 使用场景，其中不少操作信息以截图、动图和演示视频呈现。为了方便搜索、引用、版本维护和 Agent 理解，本项目会把这些视觉化内容进一步整理成文本化、结构化的 Markdown 文档。

本项目以 Agent 可读为优先，重新整理 LayaAir 3.x 的中文文档。核心产物首先是结构化 Markdown、LLM 入口和可解析的图表，然后再渲染成人类方便阅读的静态文档站。

## 我们要解决什么

- 将截图、动图或视频中的操作步骤，补充成清晰的 Markdown 文档
- 把场景、节点、组件、资源、脚本、预制体等概念关系讲清楚
- 对需要更多上下文的章节，补充前置概念、关联关系和常见边界条件
- 把流程、层级、依赖关系整理成 draw.io 等可读图表
- 让文档优先适合 Agent 检索、理解和复用，同时渲染成人类方便阅读的页面

## 写作原则

1. 先解释概念，再描述操作。
2. 能用文字说明的内容，不依赖截图代替说明。
3. 复杂关系优先补结构图、流程图或对照表，尽量使用 draw.io 等可解析格式。
4. 每篇文档尽量说明适用场景、前置条件、操作步骤和常见误区。
5. 保留 Markdown 作为主要信息源，HTML 只是面向人类的渲染结果。

## 文档形态

这个项目使用 `tiny-docs` 构建文档站，发布时会同时生成：

- 面向 Agent / LLM 使用的 `llms.txt`
- 对应页面的 Markdown 镜像
- 文档依赖的图片、draw.io、Excalidraw 等静态资源
- 面向人类阅读的静态 HTML 文档
- Pagefind 搜索索引

这意味着同一份内容会先服务机器可读和可维护，再服务浏览器阅读：

- Agent 对 LayaAir 知识的检索、理解和改写
- 文档维护者对内容结构的持续整理
- 学习 LayaAir 的开发者在浏览器中系统阅读

## 当前方向

优先围绕 LayaAir 3.4 中文文档整理可被 Agent 使用的知识结构：

- 场景、节点和组件模型
- 资源管理与引用关系
- 脚本、生命周期和运行机制
- 编辑器相关操作背后的数据关系和执行逻辑
- 项目发布与调试
- 官方资料中以截图、动图或视频呈现的重点章节

## 本地构建

```bash
yarn tiny-docs -c configs.yaml publish dist
```

构建产物输出到 `dist`。

## 发布

项目通过 GitHub Actions 发布到 GitHub Pages：

```text
https://geequlim.github.io/LayaAir-Docs/
```

